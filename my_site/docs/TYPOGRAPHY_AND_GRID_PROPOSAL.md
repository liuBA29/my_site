python# Предложение: типографика и сетка для сайта LiuBA29

Документ только с рекомендациями. Ничего в коде пока не меняется.

---

## 1. Текущее состояние (кратко)

- **Шрифты:** заголовки — Nunito, основной текст — Source Sans 3.
- **Контейнер:** `width: 90%`, `max-width: 1200px`.
- **Заголовки:** `h1`/`h2` — `1.2em`, по центру, цвет акцентный.
- **Текст:** `line-height: 1.8` у параграфов, размеры в разных блоках заданы по-разному (em, rem, px).

Сейчас нет единой шкалы размеров и отступов, из-за этого сложнее держать визуальный ритм и «современный» вид.

---

## 2. Типографика

### 2.1 Шрифты (оставить или чуть усилить)

- **Заголовки:** оставить **Nunito** — мягкий, узнаваемый, хорошо сочетается с розовой палитрой.
- **Основной текст:** оставить **Source Sans 3** — нейтральный, хорошо читается.

Опционально для «обновления» без смены характера:
- Заголовки: **Nunito Sans** (чуть более современная геометрия) или оставить Nunito.
- Текст: **Source Sans 3** уже удачный; альтернатива — **Inter** (если захочется более «нейтральный» интерфейсный вид).

Рекомендация: **ничего не менять**, только упорядочить размеры и межстрочные интервалы.

### 2.2 Шкала размеров (modular scale)

Использовать одну базу и коэффициент, чтобы все размеры были связаны. Например база **16px** и коэффициент **1.25** (Major Fourth):

| Название   | Размер (rem) | Примерно (px) | Где использовать |
|------------|--------------|---------------|-------------------|
| `--text-xs` | 0.75rem      | 12px          | Подписи, метки, мелкий текст в карточках |
| `--text-sm` | 0.875rem     | 14px          | Вторичный текст, футер, даты |
| `--text-base` | 1rem       | 16px          | Основной текст страницы |
| `--text-md` | 1.125rem     | 18px          | Лид-абзац, важный текст в блоке |
| `--text-lg` | 1.25rem      | 20px          | Подзаголовки секций (h3) |
| `--text-xl` | 1.5rem       | 24px          | Заголовки секций (h2) |
| `--text-2xl` | 1.875rem    | 30px          | Главный заголовок страницы (h1) |
| `--text-hero` | 2rem или 2.25rem | 32–36px   | Опционально: крупный заголовок на главной |

Сейчас h1/h2 заданы как `1.2em` — визуально мелковаты для «современного» акцента. Имеет смысл поднять h1 до ~1.75–2rem, h2 — до ~1.5rem, сохранив цвет и выравнивание.

### 2.3 Межстрочный интервал (line-height)

- Основной текст: **1.6–1.7** (сейчас 1.8 — можно оставить или чуть снизить для плотности).
- Заголовки: **1.2–1.35**.
- Мелкий текст (подписи, футер): **1.4–1.5**.

### 2.4 Длина строки (читаемость)

Для блоков с большим объёмом текста целевая ширина строки **45–75 символов** (примерно 28–35em при `font-size: 1rem`). Можно задать `max-width` только для контентных колонок, не для всего контейнера.

---

## 3. Сетка

### 3.1 Контейнер

- **Десктоп:** `max-width: 1200px` — оставить.
- **Ширина:** `width: 90%` или `width: min(90%, 1200px)` — ок.
- Дополнительно можно ввести **узкий контейнер для текста** (например `max-width: 65ch` или 720px) для статей и длинных описаний.

### 3.2 Колонки и отступы

- **Базовая сетка:** 12 колонок (логически; в CSS можно реализовать через flex/grid).
- **Gutter (зазор между колонками):** один фиксированный шаг, например **24px** или **1.5rem** на всех брейкпоинтах.
- **Внешние отступы контейнера:** совпадают с gutter или кратны ему (например 24px / 1.5rem по бокам на мобильных).

### 3.3 Брейкпоинты

Привести к одному набору и использовать везде:

| Имя        | Ширина    | Назначение |
|------------|-----------|------------|
| `mobile`   | до 480px  | Узкий телефон |
| `mobile-lg`| до 600px  | Широкий телефон / мелкий планшет |
| `tablet`   | до 768px  | Планшет |
| `desktop-sm` | до 1024px | Узкий ноутбук |
| `desktop`  | от 1024px | Ноутбук и шире |

Сейчас в проекте уже встречаются 375px, 480px, 600px, 768px, 900px, 1024px — можно оставить, но в документе/стилях зафиксировать «основные» (например 600, 768, 1024) и остальные считать дополнительными.

---

## 4. Шкала отступов (spacing)

Один шаг — например **8px (0.5rem)**. Все отступы и gap — кратные ему:

| Переменная   | Значение | Пример использования |
|--------------|----------|-----------------------|
| `--space-1`  | 0.25rem (4px)  | Минимальные отступы внутри кнопок, иконок |
| `--space-2`  | 0.5rem (8px)   | Между иконкой и текстом, мелкие паддинги |
| `--space-3`  | 0.75rem (12px) | Между элементами в списках, в футере |
| `--space-4`  | 1rem (16px)    | Отступ между параграфами, блоками |
| `--space-5`  | 1.5rem (24px)  | Gutter, отступ секции от контента |
| `--space-6`  | 2rem (32px)    | Отступ между секциями |
| `--space-8`  | 2.5rem (40px)  | Крупные блоки, главная |
| `--space-10` | 3rem (48px)    | Большие разделители секций |

Так проще держать ритм и «воздух» без хаотичных `20px` / `18px` / `25px` в разных местах.

---

## 5. Итоговые рекомендации (без смены кода)

1. **Шрифты:** оставить Nunito + Source Sans 3.
2. **Типографика:** ввести шкалу размеров (от `--text-xs` до `--text-2xl` или `--text-hero`) и привязать к ней все заголовки и текст.
3. **Заголовки:** поднять визуальный вес — h1 крупнее (1.75–2rem), h2 около 1.5rem, сохранив цвет и центрирование.
4. **Сетка:** зафиксировать один набор брейкпоинтов и один gutter (например 24px), контейнер оставить 90% / 1200px.
5. **Отступы:** ввести шкалу отступов (например по 8px) и по возможности заменить «магические числа» на переменные.

Если захочешь, следующий шаг — по этому документу можно точечно внедрить переменные и правила в `styles.css` и `styles_light.css`, не ломая текущий вид, а только упорядочивая его.
